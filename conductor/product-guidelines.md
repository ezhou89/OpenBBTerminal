# Product Guidelines

## Communication Style

- **Technical but Accessible**: Use precise financial terminology but explain complex concepts
- **Clear and Concise**: Avoid jargon when simpler language works
- **Educational**: Help users understand not just "what" but "why"
- **Professional**: Maintain credibility while being approachable

## UX Patterns

### Data Presentation
- Standardize output formats across providers
- Provide clear error messages with actionable guidance
- Include data freshness indicators where relevant
- Support both detailed and summary views

### Options Research (Focus Area)
- Make Greeks and risk metrics accessible to average investors
- Provide contextual explanations for complex options strategies
- Visualize options chains and payoff diagrams clearly
- Surface relevant insights without overwhelming detail

### API Design
- Consistent naming conventions across endpoints
- Predictable response structures
- Comprehensive error handling with status codes
- Rate limiting transparency

## Quality Standards

- All public APIs must have documentation
- Breaking changes require deprecation notices
- Performance regressions must be addressed immediately
- Security vulnerabilities are highest priority fixes
